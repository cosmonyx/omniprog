Rust code can be compiled in two ways:

If a Rust program has no external dependencies, it can be compiled directly 
using the Rust compiler rustc.

rustc main.rs

For most real-world projects, Rust uses Cargo, which is the official build 
system and package manager for Rust.

Cargo requires a project configuration file named Cargo.toml in the project 
directory.

How to build:

cargo build            # build in debug mode
cargo run              # build and run in debug mode
cargo build --release  # build optimized release version
cargo run --release    # run optimized release version
cargo clean            # remove all build artifacts

Cargo automatically downloads dependency source code from crates.io and 
compiles them along with the project.

crates.io is the official public repository where Rust developers publish and 
share reusable Rust crates.

It also generates Cargo.lock that records the exact versions of all 
dependencies used in the build.

About Rust:

Rust is a modern systems programming language with performance comparable 
to C and C++.

Rust was originally started in 2006 by Graydon Hoare at Mozilla to build a 
systems programming language that provides memory safety and concurrency.

Rust is currently maintained by the Rust Foundation, and Rust crates are 
published and shared through crates.io, the official repository for Rust packages.

It provides memory safety without a garbage collector through its ownership 
and borrowing model.

Rust programs are compiled using the rustc compiler. It includes Cargo, an 
integrated build system and package manager for dependency management.

Rust is used in areas such as systems software, embedded systems, networking, 
and cloud infrastructure.

Rust is being adopted in major projects including the Linux kernel for 
writing memory-safe kernel components.

Rust supports many platforms including Linux, Windows, macOS, Android, iOS, 
and embedded architectures such as ARM and RISC-V.
